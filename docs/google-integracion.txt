INTEGRACION GOOGLE CALENDAR, TASKS Y SHEETS
==========================================

Objetivo
--------
La app usa Neon como fuente de verdad. Google se conecta como destino operativo:

- Google Calendar: crea o actualiza eventos.
- Google Tasks: crea o actualiza tareas.
- Google Sheets: agrega filas con el estado de la agenda.

Archivos de codigo relevantes
-----------------------------
- src/lib/google-integration.ts
- src/app/api/integrations/google/connect/route.ts
- src/app/api/integrations/google/callback/route.ts
- src/app/api/data/agenda/integrations/route.ts
- src/app/api/data/agenda/sync/route.ts

Variables de entorno requeridas
-------------------------------
Definelas en tu entorno local, en Vercel y en cualquier runtime donde corra la app:

- GOOGLE_CLIENT_ID
- GOOGLE_CLIENT_SECRET
- GOOGLE_OAUTH_REDIRECT_URI
- GOOGLE_INTEGRATION_SECRET

Valor esperado de cada una
--------------------------
1. GOOGLE_CLIENT_ID
   Cliente OAuth 2.0 creado en Google Cloud Console.

2. GOOGLE_CLIENT_SECRET
   Secreto del mismo cliente OAuth.

3. GOOGLE_OAUTH_REDIRECT_URI
   URL publica que Google usara para devolver el codigo OAuth.
   Ejemplo productivo:
   https://recalc.relead.com.mx/api/integrations/google/callback

4. GOOGLE_INTEGRATION_SECRET
   Clave larga y privada para cifrar tokens antes de guardarlos en Neon.
   Recomendacion:
   - minimo 32 caracteres
   - aleatoria
   - no reutilizar una password humana

Configuracion en Google Cloud
-----------------------------
1. Entrar a Google Cloud Console.
2. Crear o reutilizar un proyecto.
3. Habilitar estas APIs:
   - Google Calendar API
   - Google Tasks API
   - Google Sheets API
4. Configurar OAuth consent screen.
5. Crear credenciales tipo OAuth Client ID para aplicacion web.
6. Agregar Authorized redirect URI:
   https://recalc.relead.com.mx/api/integrations/google/callback
   Si usas preview o local, agrega tambien esas rutas exactas.

Scopes usados por la app
------------------------
- https://www.googleapis.com/auth/calendar.events
- https://www.googleapis.com/auth/tasks
- https://www.googleapis.com/auth/spreadsheets
- openid
- email
- profile

Como se conecta el usuario
--------------------------
1. La app construye la URL OAuth en:
   src/lib/google-integration.ts
2. El usuario entra por:
   /api/integrations/google/connect
3. Google redirige a:
   /api/integrations/google/callback
4. La app intercambia el code por tokens.
5. Los tokens se cifran y se guardan en:
   recalc_admin.user_google_connection

Donde se guardan las preferencias de sync
-----------------------------------------
Tabla:
- recalc_admin.agenda_sync_preference

Campos clave
------------
- syncCalendarEnabled
- syncTasksEnabled
- syncSheetsEnabled
- calendarId
- tasklistId
- spreadsheetId
- worksheetName

Como configurar el destino
--------------------------
La app expone preferencias por usuario en:
- GET /api/data/agenda/integrations
- PATCH /api/data/agenda/integrations

Payload ejemplo para activar todo:

{
  "syncCalendarEnabled": true,
  "syncTasksEnabled": true,
  "syncSheetsEnabled": true,
  "calendarId": "primary",
  "tasklistId": "@default",
  "spreadsheetId": "TU_SPREADSHEET_ID",
  "worksheetName": "Agenda"
}

Que se necesita para cada servicio
----------------------------------
Google Calendar
- calendarId
- Si no defines uno, la app usa "primary".

Google Tasks
- tasklistId
- Si no defines uno, la app usa "@default".

Google Sheets
- spreadsheetId obligatorio
- worksheetName opcional, default: "Agenda"

Como se lanza la sincronizacion
-------------------------------
Endpoint:
- POST /api/data/agenda/sync

Ese endpoint:
- lee la agenda desde Neon,
- consulta la conexion Google del usuario,
- y sincroniza hacia Calendar, Tasks y/o Sheets segun la preferencia guardada.

Que NO hace esta integracion
----------------------------
- Neon no se reemplaza con Google Sheets.
- Google no es la base principal.
- No se sincronizan contactos de Google.
- No se hace polling automatico por cron todavia.

Recomendacion operativa
-----------------------
1. Conecta primero Google.
2. Activa solo un destino.
3. Prueba con 1 o 2 items.
4. Luego activa los demas destinos.
5. En produccion, agrega un boton o job controlado para sync manual o programado.

Si algo falla
-------------
1. Verifica variables de entorno.
2. Verifica redirect URI exacto.
3. Verifica que las APIs esten habilitadas.
4. Verifica que el usuario haya concedido scopes.
5. Revisa recalc_admin.user_google_connection.lastSyncError.
